## 1) 모집 상태별 코드 동작 매트릭스

**판정 로직**: `lib/event-config.ts` → `getRecruitmentStatus()`, `isRecruitmentOpen()`  
**클라이언트·API 동일 함수 사용** — 화면과 서버가 어긋나지 않음.

| 상태 | 조건 | `isRecruitmentOpen` | 랜딩 배너 | 신청 페이지 | 제출 버튼 | API POST |
|------|------|---------------------|-----------|-------------|-----------|----------|
| `tbd` | 모집 start/end가 `TBD` | `false` | "모집 일정 곧 공지" | 배너 표시 | "일정 공지 대기" (비활성) | 403 — 일정 미확정 |
| `upcoming` | 오늘 < 모집 시작 | `false` | "모집이 곧 열려요" | 배너 표시 | "모집 오픈 전" (비활성) | 403 — 오픈 전 |
| `open` | 모집 기간 내 | `true` | **배너 숨김** | 배너 없음 | "신청 완료하기" (활성) | 200 (시트 연동 성공 시) |
| `closed` | 오늘 > 모집 종료 | `false` | "이번 회차 모집 마감" | closed 배너 **숨김** | "모집 마감" (비활성) | 403 — 마감 |
| `closed` (수동) | `recruitmentManualOpen = false` | `false` | "모집 마감" | closed 배너 숨김 | "모집 마감" | 403 — 마감 |

### 일정별 예상 상태 (현재 `event-config.ts` 기준)

| 날짜 | 예상 상태 |
|------|-----------|
| 2026-06-30 (오늘) | `upcoming` |
| 2026-07-01 ~ 07-05 | `open` |
| 2026-07-06 이후 | `closed` |

### 화면별 상세 (코드 기준)

**랜딩 (`/`)**
- `RecruitmentBanner` — `open`이 아니면 표시
- `open`이 아닐 때 "신청 페이지 미리 보기" 링크 표시 (`tone !== closed`)
- CTA "오랜디런 참여하기" → `/apply` (모집 상태와 무관하게 링크는 항상 활성)

**신청 (`/apply`)**
- `isRecruitmentBlocked`일 때: 폼 필드 비활성, 안내 문구, 제출 차단
- `handleSubmit` 초반 `if (isRecruitmentBlocked) return` — API 우회 방지
- `closed` 상태: `RecruitmentBanner` context=apply 에서 배너 숨김 (폼 안내 문구만 표시)

**API (`POST /api/apply`)**
- `!isRecruitmentOpen()` → 403 + 상태별 한국어 메시지
- 환경변수 누락 → 500
- Zod 검증 실패 → 400
- 시트 웹훅 실패 → 502 (시크릿/URL/액세스별 상세 메시지)
