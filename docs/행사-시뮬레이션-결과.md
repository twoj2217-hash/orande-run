## 0) 시뮬레이션 기준과 가정

### 기준

- 일정은 브리핑의 TBD가 아니라 코드 기준으로 고정했습니다.
  - 모집: 7/1~7/5
  - 챌린지: 7/13~8/9 (4주)
  - 인증 마감: 8/16
  - 발송 안내: 챌린지 종료 후 2주 이내
- 운영 모델은 100명 이하, 시트 수동 운영, PG 미도입, 로그인 미도입 전제를 유지했습니다.

### 가정

- 참가자 규모: 30~80명
- 운영 인원: 1~2명
- 톡방 연결: 시나리오 A(미실시), 시나리오 B(일부 매칭)
- 굿즈 제작: 현재 미착수 상태에서 일정 압박이 큰 상황

## 1) 전체 평가 (5문장)

현재 구조는 100명 이하 소규모 실험 행사로는 실행 가능합니다.  
다만 "신청 완료"와 "참가 확정(입금 확인 후)"의 간극이 커서 불안 컴플레인이 집중될 가능성이 높습니다.  
가장 큰 운영 리스크는 기술이 아니라 미정 정책(입금 기한·환불·완주 기준·미완주 처리)입니다.  
모집과 인증 마감 시점에 문의가 몰리면 1~2인 수동 운영은 쉽게 병목이 발생합니다.  
모집 전까지 정책 10개를 확정하고, FAQ/고정 메시지/공지 템플릿을 준비하면 충분히 운영 가능한 구조입니다.

## 5) 긍정/부정/컴플레인/불편 항목 도출 (요약)

### 긍정 반응 (예상 12)

1. 참가비가 저렴하다.
2. 4주 분산형이라 직장인도 참여 가능하다.
3. 러닝머신/실외 모두 허용된다.
4. 50/70/100km 선택지가 직관적이다.
5. 리워드 차등(2/3/4구)이 명확하다.
6. 대전 로컬 스토리가 차별점이 된다.
7. 신청 폼 입력이 복잡하지 않다.
8. 카카오 문의 채널이 바로 보인다.
9. SNS 인증이 익숙한 사용자에게 편하다.
10. DM 대안이 있어 인스타 비공개도 참여 가능하다.
11. "혼자도 좋고 함께도 좋아요" 톤이 부담을 낮춘다.
12. 완주 후 물리 리워드가 동기부여가 된다.

### 부정 반응 (예상 12)

1. 신청 완료와 참가 확정이 다르다.
2. 입금 확인 SLA(응답 시간)가 없다.
3. 환불 기준이 없다.
4. 미입금 자동 취소 기준이 없다.
5. 완주 판정 기준이 불명확하다.
6. 미완주자 처리(리워드/환불)가 없다.
7. 톡방 연결이 실제로 보장되지 않는다.
8. "결제" 용어가 오해를 만든다.
9. 운영자 1~2명 가정에서 대응 속도 우려가 크다.
10. 굿즈 미착수 상태에서 2주 발송 약속이 공격적으로 보인다.
11. 원격 참여자에게 지역 커뮤니티 문맥이 약하다.
12. 법적 문서(처리방침/약관) 부재가 신뢰를 낮춘다.

### 예상 컴플레인 (우선순위)

- P0
  - 입금 후 24~48시간 내 확정 안내 없음
  - 완주 판정 거절 사유 미설명
  - 리워드 발송 지연 공지 부재
- P1
  - 톡방 연결 희망 제출했는데 피드백 없음
  - 중복 신청 정리 기준이 느리거나 불일치
  - 인스타 비공개 인증 가이드가 늦게 전달됨
- P2
  - 문의 응답시간 편차
  - 안내 문구 간 표현 차이(결제/입금)

### 불편 포인트 (운영 UX)

1. 참가자: 어디서 무엇을 확인해야 하는지 분산됨
2. 참가자: 인증용 DM 계정을 매번 다시 찾지 않아도 되게 해야 함
3. 참가자: 신청 후 입금 기한을 스스로 추정해야 함
4. 운영자: 시트 필터와 중복체크 수작업
5. 운영자: 문의 채널과 인증 채널이 분리됨
6. 운영자: 마감 주간에 여러 채널로 흩어지는 인증 접수 관리 부담
7. 운영자: 주소 오류/재발송 예외 처리 기준 없음

## 8) 모집 전 반드시 결정할 TOP 10

1. 입금 확인 SLA (예: 24시간 이내)  
2. 입금 기한 (예: 신청 후 48시간)  
3. 미입금 자동 취소 기준  
4. 입금 후 취소/환불 규칙  
5. 코스 변경 허용 시점과 차액 정산 방식  
6. 완주 인정 기준(기간/거리/증빙 앱)  
7. 비공개 계정 인증 표준(DM 기본 인정 여부)
8. 톡방 연결 실제 운영 여부 및 실패 시 안내 문구  
9. 발송 지연 시 공지 기준(예: 3일 초과 시 즉시 공지)  
10. 개인정보 처리방침/참가 약관 공개 여부

## 9) 즉시 적용 권장 조치 (PG/대시보드 없이)

1. 랜딩/안내 문구에서 "결제"를 "입금"으로 통일합니다.
2. 신청 완료 직후 문구에 "입금 확인 후 참가 확정"을 굵게 고정합니다.
3. 오픈채팅 고정 메시지 1개로 문의 범위·응답시간·개인정보 금지 안내를 명시합니다.
4. 인증 제출 체크리스트(필수 항목 4~5개)를 공개합니다.
5. 발송 지연 공지 템플릿을 미리 만들어 운영자 부담을 줄입니다.

## 10) 최종 판단

행사는 현재 구조로도 운영 가능하지만, 지금 상태로 오픈하면 기술 문제가 아니라 정책 미정으로 분쟁이 먼저 발생할 확률이 높습니다.  
따라서 모집 시작 전까지 TOP 10 중 P0 항목(입금·환불·완주·개인정보)부터 확정하는 것이 가장 중요합니다.
