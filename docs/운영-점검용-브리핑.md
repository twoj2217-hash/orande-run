## 점검 AI에게 주는 지시 (이 블록을 프롬프트 맨 위에 함께 붙여 넣기)

```
첨부 문서는 OranDe Run 제1회 비대면 버추얼 런 챌린지의 운영 기획입니다.
코드 리뷰·기술 스택 평가는 하지 마세요.

다음 형식으로 답해 주세요:

1. **전체 평가** (5문장): 100명 이하 규모·시트 수동 운영 전제에서 적합한지
2. **참가자 여정** 단계별 혼란·이탈 지점
3. **운영 부담** (Google Sheets 수동 운영 기준): 일일/주간 예상 업무량
4. **정책 모순** (랜딩·안내 문구 vs 실제 운영)
5. **미확정 항목 우선순위** — 모집 전 반드시 정할 것 TOP 10
6. **리스크 시나리오** (악용, 분쟁, 미입금, 미완주, 발송 지연)
7. **개선 제안** — 단, Supabase/PG/운영자 대시보드는 당장 도입하지 않는 전제
8. **타임라인 점검** — 모집·4주·인증 7일·발송 2주가 현실적인지

가정이 필요하면 "가정:"이라고 표시하고, 확정되지 않은 항목은 "미정 — 결정 필요"로 구분해 주세요.
```

## 1. 한 줄 정의

**OranDe Run (오랜디런)** — 대전 로컬 **제1회** 비대면 버추얼 런 챌린지.

- 각자 원하는 장소·시간에 **4주** 동안 코스별 거리(50/70/100km) 달리기
- 완주 인증 후 코스별 **키캡키링** 리워드 (2구 / 3구 / 4구)
- 당근 모임에서 시작된 **비공식** 커뮤니티 러닝

### 핵심 메시지 (참가자에게 전달하는 3가지)

1. **버추얼 런** — 현장 집결 없이 각자 달림
2. **누구나 참여** — 러닝머신·공원·어디서든, 실력 제한 없음
3. **리워드 = 키캡키링** — 코스별 개수 차등

### 규모·타깃 가정 (미정 항목 포함)

| 항목 | 현재 가정/상태 |
|------|----------------|
| 예상 참가자 | **미정** (100명 이하 가정) |
| 주 타깃 | 대전 로컬 + 전국 원격 참여자 |
| 운영 인원 | **미정** |
| 모집 목표 인원 | **미정** |
| 완주율 목표 | **미정** |
| 키캡키링 발송 완료 목표일 | **미정** (안내 문구: 종료 후 2주 이내) |

## 2. 확정된 운영 원칙 (의도적 결정)

100명 이하 규모 → **홈페이지 관리 기능 없이 Google Sheets로 운영**

| 하지 않음 | 대신 |
|-----------|------|
| 운영자 대시보드 | 시트 + 필터 |
| 참가자 로그인 | 없음 |
| 셀프 취소/코스 변경 UI | 카카오톡 문의 |
| 자동 중복 참여 DB 차단 | 시트에서 이름·전화 수동 확인 |
| PG 결제 | 수동 계좌 이체 |

### 결정 완료 로그

| 결정 | 내용 | 날짜 |
|------|------|------|
| 신청 방식 | 웹 폼 → API → Google Sheets + 수동 입금 확인 | 2026-06 |
| 관리 UI | 만들지 않음, 시트 운영 | 2026-06 |
| 결제 | PG 없음, 계좌 이체 | 2026-06 |
| 중복 참여 | 안내 문구 + 운영자 수동 확인 | 2026-06 |

### 현재 구현 상태 (운영 관점만)

- 랜딩 페이지 + 신청 폼 + Google Sheets 연동 **코드는 거의 완료**
- 모집을 열려면 **일정·계좌·문의처·시트 배포·환경변수** 확정 필요
- 운영 설정(일정·계좌 등) 진행률: **약 20%**
- 키캡키링 굿즈: **시안 미확정, 제작 미착수**

## 3. 전체 타임라인

### 일정 (현재 코드 값 — 대부분 미확정)

| 단계 | 코드 값 | 화면 표시 | 확정값 |
|------|---------|-----------|--------|
| 모집 시작 | `TBD` | 곧 공지 | **미정** |
| 모집 종료 | `TBD` | 곧 공지 | **미정** |
| 참여 시작일 | `TBD` | 곧 공지 | **미정** |
| 참여 기간 | 4주 | 4주 챌린지 | 확정 (변경 시 명시) |
| 완주 인증 기한 | — | 챌린지 종료 후 7일 이내 | 안내만 있음, 세부 **미정** |
| 리워드 발송 | — | 종료 후 2주 이내 발송 예정 | 세부 **미정** |

### 모집 상태 로직 (참가자가 보는 화면)

| 상태 | 조건 | 참가자에게 보이는 것 |
|------|------|----------------------|
| `tbd` | 모집 시작/종료가 TBD | "모집 일정 곧 공지" 배너 |
| `upcoming` | 오늘 < 모집 시작 | "모집이 곧 열려요" 배너 |
| `open` | 모집 기간 내 | 배너 숨김, **신청 가능** |
| `closed` | 모집 종료 또는 운영자 긴급 마감 | "이번 회차 모집 마감", 신청 403 차단 |

- 긴급 모집 중단: 운영자가 `recruitmentManualOpen = false`로 배포

### 단계별 운영 타임라인 (채워 넣을 템플릿)

| 단계 | 기간 | 참가자 행동 | 운영자 행동 | 문의 채널 |
|------|------|-------------|-------------|-----------|
| 사전 홍보 | **미정** | 랜딩 방문, 코스 비교 | 인스타·당근 등 홍보 | 카카오톡 |
| 모집 | **미정** | 신청 폼 제출 → 입금 | 시트 확인, 입금 대조, 중복 검사 | 카카오톡 |
| 모집 종료 ~ 참여 시작 | **미정** (간격 일수 미정) | 대기 | 참여 시작 안내 준비, (선택) 톡방 연결 | 카카오톡 |
| 챌린지 4주 | 참여 시작일 + 4주 | 각자 달리기 | (선택) 응원·리마인드, 톡방 운영 | 카카오톡 |
| 완주 인증 | 종료 후 7일 | 인스타 해시태그 또는 메일 | 인증 검토 | 메일/인스타 |
| 리워드 발송 | 종료 후 2주 내 (안내) | 수령 | 발주·포장·택배, 시트 발송완료 기록 | 카카오톡 |

### 미정 — 일정 관련 결정 필요

- [ ] 4주의 **캘린더 주** 정의 (월~일 기준? 참여 시작일 포함?)
- [ ] 모집 종료 후 참여 시작까지 **간격** 며칠?
- [ ] `upcoming` 상태에서 "미리 신청해 두시면" 문구 vs 실제 `open` 전 신청 불가 — **문구·동작 일치 여부 점검 필요**

## 5. 참가자 여정 (상세)

### 전체 흐름

```
랜딩 방문 → 코스 선택 → 신청 폼 → Google Sheets 기록 → 참가비 입금
→ (입금 확인 후 참가 확정) → 4주 버추얼 런 → 완주 인증 → 키캡키링 발송
```

### 랜딩에 노출되는 참여 4단계

1. 모집 기간에 코스 선택 후 참가비 **결제** ← 실제로는 **입금**
2. 참여 시작일 안내 후 4주 동안 거리 채우기
3. 완주 기록 인증 (인스타 해시태그 또는 운영진 메일)
4. 인증 확인 뒤 티어별 키캡키링 발송

### 신청 완료 후 참가자에게 보이는 5단계 안내

| 단계 | 제목 | 내용 요약 |
|------|------|-----------|
| 1 | 참가비 입금 | 안내 계좌로 입금, 확인 후 참가 확정 |
| 2 | 4주 버추얼 런 | 각자 페이스, 러닝머신·공원 어디서든 OK |
| 3 | 혼자도 좋고, 함께도 좋아요 | (선택) 비슷한 동네·시간대 3~4명 규모 톡방 연결 가능 |
| 4 | 완주 인증 | SNS 해시태그 또는 메일로 기록 제출 |
| 5 | 키캡키링 발송 | 인증 확인 후 코스별 키캡키링 발송 |

### 신청 폼 수집 항목

| 필드 | 필수 | 용도 |
|------|------|------|
| 코스 (50k/70k/100k) | ✅ | 참가비·리워드 결정 |
| 지역 (대전 5구 / 대전 외) | ✅ | `대전/서구` 형식 저장, 톡방 연결 참고 |
| 이름 | ✅ | 입금 대조·발송 |
| 휴대폰 | ✅ | 중복 확인·연락 |
| 이메일 | 선택 | 연락 |
| 선호 요일 | 선택 | 톡방·안내 참고 (미입력 OK) |
| 선호 시간 | 선택 | 톡방·안내 참고 (미입력 OK) |
| 개인정보 동의 | ✅ | — |

**개인정보 동의 요약**: 이름, 연락처, 지역, 러닝 계획(선택) 수집·참가 안내·운영 목적, **챌린지 종료 후 1년 보관**

### 신청 → 시트 흐름 (운영 관점 요약)

1. 참가자가 `/apply`에서 제출
2. 모집 `closed`면 거부 (403)
3. 검증 통과 시 Google Sheets에 행 추가 (웹훅)
4. 참가자에게 완료 모달: 입금 안내, 계좌 복사, 진행 가이드, 인증 안내

**시트 기본 열**: 제출시각, 이름, 휴대폰, 이메일, 코스, 참가비, 입금확인(수동)

**추가 전송·권장 열**: 지역, 선호 요일, 선호 시간, 중복체크 메모, 톡방 연결, 발송완료

## 6. 정책 상세

### 참가 정책 (안내 문구 — 자동 집행 없음)

- 1인 1코스만 참여 (예: 50km 신청 후 70km·100km 추가 신청 불가)
- 신청 후 취소·코스 변경은 **카카오톡** 문의
- 입금 확인 전 문의 시 변경·취소가 수월하다고 안내

### 환불·취소 (대부분 미정)

| 상황 | 현재 정책 |
|------|-----------|
| 입금 전 취소 | **미정** (전액 환불 가정 가능) |
| 입금 후 취소 | **미정** |
| 코스 변경 | **미정** (입금 전만 / 입금 후도 / 불가) |
| 입금 기한 | **미정** (신청 후 N시간/일) |
| 미입금 시 자동 취소 | **없음** (현재 수동 처리) |
| 입금자명 ≠ 신청자명 | 운영자 확인 연락 (기한·미응답 처리 **미정**) |

## 9. 알려진 모순·표현 이슈 (점검 시 우선 확인)

| 이슈 | 설명 |
|------|------|
| "결제" vs "입금" | 랜딩 4단계·참여 절차에 "결제" 표현, 실제는 계좌 이체 |
| 신청 vs 입금 확정 | 신청 완료 ≠ 참가 확정. "입금 확인 후 참가 확정"은 완료 모달에만 상세 |
| 자동 vs 수동 | 1인1코스·중복·입금 확인 모두 수동 — 참가자는 안내만 봄 |
| 톡방 연결 | 랜딩·완료 모달에 안내 있으나 실제 제공 여부 **미정** |
| 일정 TBD | 모집 상태 `tbd`에서도 사이트 열람 가능, 신청은 일정 확정 전 제한적 |
